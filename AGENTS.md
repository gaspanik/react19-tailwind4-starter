# Project Guidelines

## Stack and Structure

- React 19, TypeScript, Vite 8, TanStack Router, Tailwind CSS v4, and Biome.
- Entry flow is `index.html` -> `src/main.tsx` -> `src/routes/__root.tsx`.
- Routes live in `src/routes/`. Use TanStack Router's `createFileRoute()` pattern.
- `src/routeTree.gen.ts` is auto-generated by the router plugin. Never edit it manually.
- Reusable UI belongs in `src/components/`. Shared utilities belong in `src/lib/`.
- Use the `@/` alias for imports from `src/`.

## Build and Validation

- Replace `<pm>` with your package manager (pnpm / bun / yarn / npm). pnpm is the default.
- Install dependencies with `<pm> install`.
- Start the dev server with `<pm> run dev`.
- Validate production builds with `<pm> run build`.
- Run linting and formatting with `<pm> run check`.

## Code Style

- Use the React JSX transform. Do not add `import React from 'react'`.
- Import React hooks as named imports.
- Prefer semantic HTML and explicit TypeScript props/interfaces.
- Keep changes consistent with the existing file layout and naming.
- Use `cn()` from `src/lib/utils.ts` for Tailwind class composition.
- Use `lucide-react` for icons.

## Tailwind Conventions

- Tailwind CSS v4 is configured through `src/index.css` and `@tailwindcss/vite`.
- Do not add a `tailwind.config.js` file.
- Keep styling in `className` props. Do not introduce CSS modules or styled-components.
- Prefer Tailwind's standard spacing scale before arbitrary values.
- Do not use deprecated v3 patterns such as `space-x-*`, `space-y-*`, or `divide-*`; use flex/grid layouts with `gap-*` and explicit borders instead.
- Put reusable theme tokens in `src/index.css` with `@theme` when a value repeats.
- `--heading-font-family` is applied to h1–h6 via `@layer base`. `--default-font-family` is the body default. Never use `font-[var(--heading-font-family)]`, `font-(--heading-font-family)`, or their `--default-font-family` equivalents — they are redundant. To apply the heading font on a non-heading element, add the selector to `@layer base` or define a dedicated `@theme` utility.

## Project Conventions

- Keep images in `src/assets/images/`.
- Use `src/lib/image.ts` for eager image access and `src/lib/imageAsync.ts` for lazy image access.
- Add new routes by creating files under `src/routes/`; let the router regenerate the tree.
- Provide meaningful `alt` text for images and accessible labels for navigation and interactive controls.
